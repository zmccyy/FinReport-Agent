# M1.12 文件上传接口实现决策记录

> **日期**：2026-07-15
> **任务**：M1.12 L2 文件上传接口 /reports/upload
> **状态**：已完成

---

## 背景

实现 `POST /api/v1/reports/upload` 端点，支持 PDF 财报上传到 MinIO，创建 report + task 记录，支持幂等性和 MD5 去重。

---

## 决策列表

### D1: 上传流程顺序：先创建 Task → 上传 MinIO → 创建 Report → 调度 PARSE

**初始问题**：TaskOrchestrator.createTask 会立即调度 PARSE 步骤（发送 MQ 消息），但 MinIO 上传在之后执行。L3 消费者可能在 PDF 上传前就收到 PARSE 消息。

**决策**：新增 `createTaskWithoutDispatch` 方法（只创建 task 记录和 step 记录，不调度），上传 MinIO 成功后调用 `dispatchTask(taskId, extraPayload)` 调度 PARSE，并将 `pdfObjectKey` 加入消息 payload。

### D2: 幂等性实现

**决策**：使用 Redis `fin:idem:upload:{key}` 键缓存 UploadResponse（JSON），TTL 24 小时。缓存失败（Redis 不可用/序列化失败）降级放行，不影响主流程。

### D3: MD5 去重

**决策**：基于 report 表的 `UNIQUE KEY uk_md5 (pdf_md5)` 约束。同一文件重传时复用已有 report 记录，创建新 task 关联。

### D4: 错误恢复

**决策**：MinIO/DB 操作失败时，调用 `TaskOrchestrator.markTaskFailed()` 标记 task 为 FAILED，避免孤儿任务。

### D5: TaskOrchestrator 新增方法

- `createTask(Long userId, Long refReportId, Map payload)` — 支持带 refReportId 的任务创建
- `createTaskWithoutDispatch(Long userId, Long refReportId, Map payload)` — 创建任务但不调度 PARSE
- `dispatchTask(String taskId, Map extraPayload)` — 手动调度 PARSE，合并额外 payload
- `updateRefReportId(String taskId, Long reportId)` — 更新任务的报告关联
- `markTaskFailed(String taskId, String errorMsg)` — 标记任务失败

### D6: FileService 重构点

- 使用 `DataBufferUtils.join().handle()` 确保 `DataBufferUtils.release()` 在 finally 块中执行
- Content-Type 验证使用 `MediaType.isCompatibleWith()` 支持带参数的 MIME
- MD5 计算使用 `BusinessException` 而非 `RuntimeException`
- MinIO `makeBucket()` 捕获 `BucketAlreadyOwnedByYou` 处理并发创建

## 代码审查发现的关键问题与修复

| # | 严重度 | 问题 | 修复 |
|---|--------|------|------|
| 1 | CRITICAL | PARSE 在 MinIO 上传前调度，L3 找不到文件 | 改为 createTaskWithoutDispatch + dispatchTask |
| 2 | CRITICAL | 消息 payload 缺少 pdfObjectKey | dispatchTask 支持 extraPayload |
| 3 | HIGH | ensureBucketExists 并发竞态 | 捕获 BucketAlreadyOwnedByYou |
| 4 | HIGH | 幂等缓存失败导致上传失败 | onErrorResume 降级 |
| 5 | MEDIUM | Redis 不可用时幂等检查失败上传 | checkIdempotency 加 onErrorResume |
| 6 | MEDIUM | DataBuffer 泄漏（OOM 路径） | handle() + try/finally |
| 7 | LOW | Content-Type 参数导致误拒 | isCompatibleWith 代替 equals |
| 8 | LOW | RuntimeException 违反 CLAUDE.md | 改为 BusinessException |
