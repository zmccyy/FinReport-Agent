## 关键决策

1. **SSE 客户端用 fetch + ReadableStream 而非 EventSource**
   后端 `/tasks/{id}/stream` 由 JwtFilter 强制 `Authorization: Bearer` 校验，
   而原生 EventSource 无法携带自定义请求头。自研 SSE 解析器（SseParser）支持：
   Authorization 头、Last-Event-ID 重连（服务端 Redis 回放）、指数退避（1s 基数
   ×2^n，最多 5 次）、401 时单飞刷新 token 后重连一次、终态（done/error）后不再重连。
   备选方案 `fetch-event-source` 库被排除：CLAUDE.md §9.2 禁止引入设计文档未提及的新依赖。

2. **token 存储独立成 api/token.ts 模块**
   打破 http.ts（拦截器需读 token）与 auth store（登录需写 token）的循环依赖。

3. **401 处理：单飞（single-flight）刷新 + WeakSet 防重试循环**
   并发多个 401 共享同一次 refresh 请求；每个请求只重试一次，refresh 端点自身
   不参与重试（防递归）。

4. **UI 阶段聚合**：后端 6 步骤（PARSE/EXTRACT_BS/IS/CF/CHECK/REPORT）在进度卡片
   聚合为 4 阶段，EXTRACT 三表以标签明细展开。进度百分比取事件 progress 最大值
   （单调不回退），对齐 orchestrator 常量（15/55/75/100）。

5. **Reports 列表 M1 过渡方案**：后端尚无 GET /reports 列表接口（M2+），
   前端用 localStorage 持久化本用户上传记录，标注 ⚠️ 待列表接口上线后改为后端拉取。

6. **tsconfig 重构为 create-vue 标准三文件**（solution + app + node）：
   原先单文件 references tsconfig.node.json 缺 composite 触发 TS6306/TS6310。
   type-check 脚本改为 `vue-tsc -b --force`。

## 发现并修复的 Bug

| # | Bug | 根因 | 修复 |
|---|---|---|---|
| 1 | 旧 sse.ts 监听 `complete` 事件 | 后端实际发 `done` 事件，旧代码永远收不到完成通知 | 重写为 done/error/progress 三类路由 |
| 2 | 注册/登录后不跳转，永远停在 /login | `isAuthenticated = computed(() => hasToken())` 读 localStorage，非 Vue 响应式源，首算 false 后永不更新，路由守卫始终拦截 | 改为手动同步 ref，setTokens/clearTokens 后调 syncAuthState()（commit 1cdd1ed） |
| 3 | el-upload limit=1 时无法替换文件 | 默认 exceed 行为是拒绝新文件 | on-exceed 中 clearFiles + handleStart 实现替换 |
