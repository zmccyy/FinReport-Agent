## M3.01 交付说明（2026-07-21）

### 实现文件

- `ai-service/app/schemas/reasoning.py` — 新增 schemas：
  - `RuleType` 三条硬编码勾稽规则枚举
  - `Severity` 严重度（INFO / WARN / ERROR / CRITICAL）
  - `RuleResult` 单条规则结果（对应 `accounting_check` 表）
  - `Anomaly` 异常检测结果（对应 `anomaly` 表，M3.03 填充）
  - `StatementSnapshot` 规则引擎入参，封装三表 + `require` 同义词查询
  - `CheckResult` Reasoner 整体输出（spec §2.3 M8 L213）
- `ai-service/app/modules/reasoner/accounting_rules.py` — 3 条规则实现：
  - `BalanceSheetIdentityRule` 资产=负债+权益（含 4 个权益同义词）
  - `NetIncomeToRetainedEarningsRule` 净利润→未分配利润（含盈余公积/应付股利可选调整项）
  - `CashFlowVsNetIncomeRule` 经营现金流 vs 净利润（含折旧摊销可选调整项）
- `ai-service/app/modules/reasoner/rule_engine.py` — `RuleEngine` 调度 3 条规则 + 计算 confidence + 异常降级
- `ai-service/tests/test_m3_rule_engine.py` — 20 个测试用例

### 关键设计决策

1. **Decimal 精度** — 所有数值用 `decimal.Decimal` 计算，避免 float 累积误差（spec §8.4）
2. **同义词组查询** — `StatementSnapshot.require(synonyms)` 任一命中即返回；缺失只回填代表名
3. **容差分级** — 默认绝对容差 0.01 元；规则 2/3 缺调整项时放宽到净利润的 5% / 20%
4. **缺失降级** — 必需科目缺失直接 `severity=CRITICAL`，`missing_items` 填充便于 M3.02 LLM 复核定位
5. **规则异常隔离** — `RuleEngine.check` 用 try/except 包裹单条规则，异常转 CRITICAL 不阻断其他规则
6. **confidence 计算** — 基础分 = 通过规则数 / 总数；每条 CRITICAL 额外扣 0.2；最低 0.0 最高 1.0
